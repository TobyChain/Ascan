**修复日期**: 2026-02-03 18:40

## 🐛 问题描述

### 症状
- 论文摘要显示为灰色背景、斜体文字（引用块样式）
- 使用了 `>` 符号创建Markdown引用块
- 视觉效果不符合简洁清晰的要求

### 根本原因
```python
# 问题代码（第699行）
st.markdown(f"> {display_abs}")
```

`>` 符号在Markdown中表示引用块（blockquote），导致：
1. 左侧边框线
2. 灰色/深色背景
3. 斜体文字
4. 内边距增加

---

## 🔧 修复方案

### 1. 修改代码显示方式
**文件**: `web/app.py`  
**行号**: 约699行

**修改前**:
```python
if display_abs:
    st.markdown("#### 📝 摘要")
    st.markdown(f"> {display_abs}")  # 引用块格式
else:
    st.info("暂无摘要信息")
```

**修改后**:
```python
if display_abs:
    st.markdown("#### 📝 摘要")
    st.markdown(display_abs)  # 普通文本格式
else:
    st.info("暂无摘要信息")
```

### 2. 简化CSS样式
**文件**: `web/app.py`  
**CSS样式部分**

**修改前**:
```css
blockquote {
    border-left: 4px solid #1f77b4;
    padding-left: 1rem;
    color: #b0b3b8;
    font-style: italic;          /* 斜体 */
    margin: 1rem 0;
    background-color: #2d3139;   /* 深色背景 */
}
```

**修改后**:
```css
blockquote {
    border-left: 4px solid #1f77b4;
    padding-left: 1rem;
    color: #e8eaed;              /* 亮色文字 */
    margin: 1rem 0;
    background-color: transparent; /* 透明背景 */
}
```
